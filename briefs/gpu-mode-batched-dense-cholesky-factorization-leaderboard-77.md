## 3. One-block-per-matrix register/shared-resident kernel for the tiny-n high-batch regime · strong

**Approach.** For {4096,n=32},{1024,n=64},{256,n=128}: assign exactly one CUDA thread block per matrix, read the matrix column-by-column for coalesced global loads, keep the whole triangle shared/register-resident, and do the entire unblocked Cholesky (right-looking rank-1 updates via warp-level primitives) without ever spilling to global memory. Write L back once. For n=32 pack multiple matrices per block to fill occupancy. This regime is pure memory-bandwidth bound: the win is single-load, single-store, zero kernel-launch-per-step overhead.

**Why it fits.** Directly the MAGMA batched-tiny recipe, and it matches the researcher's roofline instinct: these shapes each hold 16 MiB of input, so they're bandwidth-limited and the entire game is minimizing global traffic and launch overhead. The CUDA course and WebGPU-puzzles saves are exactly the shared-memory-tiling + warp-primitive skills this needs.

## 4. Software-pipelined trailing-update mainloop for the single huge matrix (n=8192..32768) · promising

## 6. Fused POTF2+TRSM single-kernel for the medium batched band (n=256..512) to kill launch overhead · promising

**Approach.** For {64,n=256},{16,n=512},{640,n=512}: the matrices are too big to be fully register-resident but small enough that per-step kernel launches (separate POTF2, TRSM, SYRK launches per block-column) dominate. Merge the panel POTF2 and TRSM into a single fused kernel per block-step so the block reads its panel once, factors and triangular-solves in shared memory, and only the trailing SYRK crosses to a second kernel. This is the regime where fusion, not raw FLOPs, wins — recursive-blocked fused kernels hit ~80% of GEMM peak per the Dongarra result.

**Why it fits.** The researcher's corpus explicitly values fusion and cache/launch-overhead minimization — they endorsed prefix-cache-friendly loops to keep repeated work cheap and annotated the offset-read trick as avoiding reading the whole thing at once. Same instinct at the kernel level: read the panel once, fuse the dependent ops, cross to global only for the trailing update. This band is the least glamorous but geometric-mean scoring makes it as valuable as any other entry.
